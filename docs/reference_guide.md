# ISO C Reference Guide

This project follows a strict requirement that every test must reference a specific paragraph of the ISO C standard.

## Format
Use the section and paragraph number, for example:
`ISO C11 §6.7.2.1p8`

- `§6.7.2.1`: Section number
- `p8`: Paragraph number

## Common Sections
- `§6.2.x`: Concepts (types, storage, linkage)
- `§6.3.x`: Conversions
- `§6.4.x`: Lexical elements
- `§6.5.x`: Expressions
- `§6.7.x`: Declarations
- `§6.10.x`: Preprocessing directives

## Why References Matter
Ensuring each test is tied to a standard paragraph makes the suite educational and verifiable. It provides the rationale for why a compiler should behave in a certain way.
